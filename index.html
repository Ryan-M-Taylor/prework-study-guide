HTML

  • An element is a type of content on a webpage.
  • Common types of elements to add content to a page include text, image, header, and list.
  • Common element types that are used to divide and describe a page include meta-related, content division, and semantic.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • "ul" creates an unordered list
  • "li" creates bullet points

CSS

  • When a developer wants to style an element directly in the HTML file they use inline CSS.
  • An internal style sheet contains CSS rules for the webpage in the head section of the HTML file.
  • An external CSS style sheet allows developers to keep all their CSS rules in a separate file.
  • When using an external style sheet, we link the file using a "link" element placed in the HTML's head.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • The "rel" attribute specifies the relationship between the current document and the linked document resource.
  • The "href" attribute specifies the location (URL) of the external resource.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • cd name-of-file: changes directory
  • ls name-of-file: check what is in current file
  • touch style.css: creates CSS style sheet file
  • touch script.js: creates a JavaScript file
  • git add -A: stage changes
  • git commit -m "description of changes": commit changes
  • git pull origin main: makes current branch up to date
  • git push origin branch-name: pushes changes

JavaScript

  • JavaScript is read from top to bottom, and the order matters.
  • A variable is a named container that allows us to store data in our code.
  • var keyword = : declares the keyword as a variable
  • Control flow is the order in which a computer executes code in a script.
  • Right click and inspect page to open the Chrome DevTools.
  • console.log : allows developers to debug and test the code that they wrote
  • if, else if : are conditional statements to introduce decision-making into the code
  • A value is considered falsy if it one of the following values : 0, -0, On, "", null, undefined, NaN, or a Boolean type with the value of false. All other values, including a Boolean with the value of true, are considered truthy.
  • "===" : strict equality operator, checks to see if two values are equal. Returns true if equal, false if not equal.
  • An array is a single variable that is used to hold a group of data. The first item in an array has an index of 0.
  • A "for" loop used the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • A function is a simple set of instructions that tells the computer how to perform a task. A function must be called in order to execute it.
